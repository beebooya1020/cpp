# 실습과제 1

## 라이브러리에는 헤더파일과 목적파일로 구성되어있다. 헤더파일과 목적파일에 들어있는 내용이 무엇인지 각각 설명하시오.
- 헤더파일은 함수, 클래스, 변수 등의 선언이 들어있는 파일로 프로그램에서 사용할 인터페이스 역할을 한다.
- 목적파일은 함수와 클래스의 실제 구현이 컴파일된 파일이며, 실행파일을 만들 때 링크되어 사용된다.

## 함수의 선언 정의 호출은 각각 어디에 존재하는가
- 선언 : 보통 헤더파일에 존재한다.
- 정의 : 실제 코드가 작성된 소스파일에 존재한다.
- 호출 : 함수를 사용하는 main 함수나 다른 소스파일에 존재한다.

## <iostream> 헤더파일에 cin, cout은 extern 변수로 선언되어있다. extern 선언과 변수선언의 차이를 설명하라. 그리고, 그럼 cin, cout 의 변수선언은 어디에 있늑나.
- extern 선언은 변수가 다른 파일에 정의되어 있음을 알리는 선언이며 메모리를 생성하지 않는다.
- 변수 선언은 실제로 메모리를 할당하여 변수를 생성하는 것이다.
- cin과 cout의 실제 정의는 C++ 표준 라이브러리의 iostream 구현 파일에 존재한다.

## 라이브러리 함수나 객체를 사용하기정네 헤더파일을 include 하는 이유를 설명하라
- 컴파일러가 함수나 객체의 이름, 타입, 매개변수 등의 선언 정보를 알 수 있도록 하기 위해서 헤더파일을 include 한다.
