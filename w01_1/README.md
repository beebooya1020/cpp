## C++ 객체지향 특성
1. 캡슐화 : 데이터와 함수를 하나로 묶고 내부 구현을 외부에서 직접 접근하지 못하도록 숨기는 기법이다.
2. 상속성 : 기존 클래스의 속성과 기능을 새로운 클래스가 물려받아 재사용하고 확장할 수 있는 특성이다.
3. 다형성 : 같은 이름의 함수나 인터페이스가 상황이나 객체에 따라 서로 다른 동작을 수행하는 특성이다.

# 알고리즘 설계와 코딩 중 무엇이 중요한가
- 알고리즘 설계가 더 중요하다. 알고리즘은 문제 해결 방법과 절차를 결정하며 프로그램의 효율성과 정확성을 좌우한다. 알고리즘이 잘 설계되면 코딩은 비교적 쉽게 구현할 수 있다.


# 클래스와 객체
- 클래스는 객체를 만들기 위한 설계도이며 데이터와 함수를 하나로 묶어 정의한 사용자 정의 자료형이다. 객체는 클래스를 기반으로 실제 메모리에 생성되어 동작하는 실체이다.


# 절차지향 프로그래밍과 객체지향 프로그래밍의 차이
- 절차지향 프로그래밍은 프로그램을 함수와 실행 순서 중심으로 구성하는 방식이며 데이터와 함수가 분리되어 있다.  
- 객체지향 프로그래밍은 데이터와 함수를 객체로 묶어 클래스와 객체 중심으로 프로그램을 구성하는 방식이다.

# 라이브러리란 무엇인가
- 라이브러리는 프로그램 개발에 필요한 함수나 클래스 등을 미리 만들어 모아둔 코드의 집합이다.

# 라이브러리가 목적파일 형태로 존재하는 이유
- 라이브러리를 목적파일 형태로 제공하면 소스 코드를 공개하지 않고도 기능을 사용할 수 있고 재사용이 용이하다.

# 링크 과정이 필요한 이유
- 링크 과정은 여러 목적파일과 라이브러리를 하나의 실행 파일로 결합하는 과정이다
